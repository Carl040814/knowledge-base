# [Agent 智能体](https://aiweb3.school/zh/handbook/ai/agent)

> 约 6 分钟阅读 · 2026-05-12

**核心定义**：Agent 是能围绕目标持续调用工具、读取状态、调整步骤的 AI 系统。关键不在"像人"，而在于能不能在明确权限和可审计流程里把建议推进到行动。

**核心分工**：模型负责提出候选行动 → 系统负责限制行动空间 → 用户负责批准高风险边界。

**第一性原理**：一个 Agent 如果只有模型和工具，就像一个会说话的脚本执行器。真正可用的 Agent 必须知道自己能做什么、不能做什么、做完如何验证、失败如何停止、谁能审计它做过的事。

**知识节点**：

| 节点             | 解释                                                                                  |
| -------------- | ----------------------------------------------------------------------------------- |
| **Tool Use**   | Agent 调用外部能力（搜索、API、代码执行、支付接口等）。工具设计要明确：输入 schema、权限范围、只读还是写入、外部副作用、调用前后记录、哪些需要人工确认 |
| **Planning**   | 把目标拆成步骤。每一步需暴露：用什么工具、读还是写、能否自动执行、是否需要用户确认、失败可否重试                                    |
| **MCP**        | 协议化工具接入方式——详见 [MCP 章节](mcp.md)                                                |
| **Skills**     | 可复用高层指令集，支持自动发现和动态生成                                                                |
| **Tracing**    | 可视化 agent 执行链——每一步调用什么工具、输入输出、耗时、谁触发的                                               |
| **Guardrails** | 输入输出验证规则，不符合则中止——详见 [Evaluation 章节](evaluation.md)                                 |
| **Handoff**    | 子任务完成后移交控制权                                                                         |
| **错误恢复**       | 执行失败时的重试、回退和人工介入                                                                    |
| **长期记忆**       | 跨 session 存储与召回——详见 [Context 章节](context.md)                                      |

**Agent 三大支柱**：
1. **权限分级**：读数据 vs 写数据库 vs 发送请求 vs 改配置 —— 不同风险等级
2. **可观测性**：任务进度、工具结果、失败原因、用户确认都要记录
3. **停止条件**：达到目标、超出预算、信息不足、风险越界、用户拒绝都应停下来

**什么时候用 Agent？** 目标开放、需多工具协作、中间结果决定下一步、需跨会话记忆。
**什么时候不用？** 一次性问答用 Prompt、流程固定用脚本、高合规要求用人工审核节点。

**Agent 失败模式**：
- **循环卡住**：不断重复同一工具调用
- **工具滥用**：不必要的或越权的工具调用
- **幻觉传递**：一个工具的幻觉传给下一个工具
- **过度信任自身输出**：把之前的输出当权威来源
- **权限蔓延**：越执行权限越高

**Prompt → Workflow → Agent 三层次对比**：

| 层次 | 决策者 | 风险 | 适用场景 |
|------|--------|------|---------|
| **Prompt** | 人在决策 | 低 | 一次性问答、翻译、总结、格式转换 |
| **Workflow** | 预定义流程 | 中 | 固定步骤的任务链，每步可预测 |
| **Agent** | 模型自主规划 + 工具调用 | 高 | 目标开放、多工具协作、中间结果决定下一步 |

> 核心判断：**谁在决策**决定风险等级。能用人来做最终判断的，不要交给 Agent。
