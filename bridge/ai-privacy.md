# [AI 隐私（AI Privacy）](https://aiweb3.school/zh/handbook/bridge/ai-privacy)

> 约 6 分钟阅读 · 2026-05-12

**核心原则**：AI 系统处理用户数据时，隐私不是功能开关，而是数据如何收集、存储、传输、使用和删除的完整设计。

## 第一性原理

Web3 场景中，链上数据公开 + AI 上下文敏感 = 隐私风险叠加。地址本身就是公开的，加上 Agent 记忆和对话历史，画像远比想象中精确。

- **最小数据原则**：只收集完成任务必需的数据
- **本地优先**：敏感处理尽量在用户侧完成
- **用户可见可控**：用户能看到什么数据被用、能撤销、能删除

## 知识节点

| 节点 | 说明 |
|------|------|
| **Data Minimization** | 只拿任务必需数据。查余额不需要交易历史，查 allowance 不需要完整地址簿。Agent 请求数据时先问：这个字段真的需要吗 |
| **Local Processing** | 敏感任务在本地执行：钱包历史摘要、私有文档分析、权限检查。本地模型做脱敏筛选，云端只收必要摘要 |
| **Data Lifecycle** | 数据从收集到删除的全过程：收集时告知、存储时加密、使用后清理。链上公开数据无法删除，但 Agent 记忆和对话可以 |
| **User Consent** | 同意连接钱包 ≠ 同意分析全部历史。分动作授权：读余额、分析交易、保存记忆、发给云端模型——各是各的开关。Consent 可撤销 |
| **Model Privacy** | 发给模型的输入可能被服务方记录。敏感场景用本地模型、脱敏输入、TEE 执行。区分"模型需要什么"和"平台收集什么" |
| **Chain-Public Data Risk** | 链上地址公开 + Agent 记忆 + 聊天 = 可组合出真人身份。产品需评估组合风险 |

## 在 AI × Web3 中的位置

AI Privacy 是 Agent Wallet、Chain-aware Context、Governance AI 的基础边界。Agent 越懂用户，泄漏后伤害越大。

## 最小实践

画 Agent 数据流图：列出输入/钱包地址/交易历史/API key/模型上下文/日志/记忆→标记每项存储位置→标记哪些发给第三方→标记可脱敏/本地处理项→写出用户如何查看/撤销/删除。
