# [AI 主权（AI Sovereignty）](https://aiweb3.school/zh/handbook/bridge/ai-sovereignty)

> 约 8 分钟阅读 · 2026-05-12

**核心原则**：用户、开发者能否控制自己的数据、模型选择、记忆、工具权限和执行环境，而不是被单一平台锁定。

## 第一性原理

越靠近用户决策和资产的 AI，越不能只依赖平台承诺。需要可检查、可撤销、可迁移的机制。

- **控制权要在用户侧可见**：用户知道哪些数据/权限/记忆被使用
- **选择权要真实存在**：模型、钱包、工具和存储不应被单点锁死
- **可验证性要保底**：关键行为有日志、签名、证明或链上记录

## 知识节点

| 节点 | 说明 |
|------|------|
| **User Control** | 用户可见/可修改/可撤销 Agent 的权限和数据。控制面板：连接的钱包、session key、授权额度、启用的工具、记忆开关、数据导出、紧急停止 |
| **Data Portability** | 偏好/记忆/任务历史/凭证可导出带走。分层：任务历史、用户偏好、工具日志、身份凭证、公有声誉、私有记忆分别导出 |
| **Model Choice** | 按任务选模型：隐私用本地、高难用强模型、低成本用小模型。系统允策略化选择，换模型需进评估流程 |
| **Local-first AI** | 敏感数据优先本地处理，必要时才调云端。混合架构：本地模型过滤脱敏→云端强模型收摘要→最终交易仍由钱包确认 |
| **Censorship Resistance** | AI 服务/模型/数据/Agent 身份不被单点封锁。开源客户端、多模型 fallback、可自托管 endpoint、可迁移身份、去中心化存储 |
| **d/acc** | Defensive Accelerationism：加速防御性/去中心化技术。AI×Web3 不只做"更强的自动代理"，还做"更好的防御工具" |
| **CROPS** | Ethereum Foundation 核心属性：Censorship resistance、Open source、Privacy、Security。AI×Web3 产品价值检查表 |

## 在 AI × Web3 中的位置

AI Sovereignty 是整条路线的价值底层。没有主权设计，AI×Web3 容易变成"用链上资产喂给更中心化的 AI 平台"。

## 最小实践

Agent 主权检查表：能导出聊天/记忆/任务/日志吗？能切换模型吗？能查看撤销 wallet/session key 吗？身份跨平台可验证吗？关键执行有审计日志吗？
