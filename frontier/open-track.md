# [开放赛道（Open Track）](https://aiweb3.school/zh/handbook/tracks/open-track)

> 约 6 分钟阅读 · 2026-05-12

**核心原则**：开放赛道适合无法被单一方向完全覆盖、但能把 AI 和 Web3 结合出新工作流/协议/体验的项目。每个项目必须回答：AI 承担什么不可替代的工作？Web3 提供什么不可替代的能力？

## 第一性原理

核心不是追热点，而是发现新的用户工作流。

- **先找工作流**：用户现在如何完成任务，哪里慢/贵/难/不透明/不可验证
- **再拆职责**：AI 负责理解/生成/规划/推荐；Web3 负责资产/身份/权限/记录/激励/治理
- **最后做最小闭环**：输入→处理→输出→权限→失败路径都要讲清楚

## 知识节点

| 节点 | 说明 |
|------|------|
| **AI-native Wallet** | 钱包理解用户目标、解释交易、管理权限、提醒风险。安全边界：AI 解释和建议，签名/授权/高风险操作交钱包权限系统 |
| **Creator / Content Collaboration** | AI 生成草稿→社区筛选二创→链上记录版本/贡献者/授权→NFT/凭证作访问权/权益→收益按贡献自动分配。坑：版权和授权 |
| **On-chain Data Analysis Agent** | 查询交易/事件/余额/协议状态→翻译成结构化解释→区分事实/推断/不确定性→给来源链接→异常行为风险提示 |
| **Cross-track Combination** | 组合方向：AI wallet+permission、Agentic commerce+settlement、Governance+source-preserving summary、Dev tooling+transaction interpreter。选一个核心闭环 |
| **New Protocol / New Scenario** | 从具体摩擦出发：现有协议为什么解决不了→AI 能给什么新能力→Web3 能给什么新保障→最小可行设计 |
