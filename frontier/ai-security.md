# [AI 安全（AI Security - 前沿）](https://aiweb3.school/zh/handbook/tracks/ai-security)

> 约 6 分钟阅读 · 2026-05-12

**核心原则**：当 Agent 开始读文档、调工具、连钱包、发交易，安全问题不只是"模型会不会答错"——是它能看到什么、能调用什么、出错后谁能发现。

## 第一性原理

安全 Agent 不是"更听话的模型"，而是被放进更小、更清楚、更可审计的行动空间。

- **上下文要分层**：系统规则、用户目标、外部资料和工具返回值不能混成同一种权限
- **工具要最小授权**：读、写、签名、转账、部署是不同风险等级
- **高风险动作要可审计**：能回放 Agent 读了什么、调用了什么、为什么执行或拒绝

## 知识节点

| 节点 | 说明 |
|------|------|
| **Agent Threat Model** | 画清危险路径：Agent 可读什么上下文、可调什么工具、工具背后连接什么账号/资金、哪些输出只是建议哪些会执行、用户在哪确认、出错能否追踪原因 |
| **Prompt Injection Defense** | 结构隔离：外部资料只能作 data 不能覆盖系统规则、检索结果带来源和可信等级、工具调用前查用户目标+权限、高风险指令须重新确认、输出经结构化校验 |
| **Tool Permission Isolation** | 按风险分层：公开只读→用户只读→草稿工具（生成不执行）→写入工具（发布/配置）→资产工具（签名/授权/转账）。越靠后越需白名单+额度+模拟+确认 |
| **Malicious Document Demo** | 安全入门最佳练习：准备含恶意指令的文档→观察 Agent 是否标记为外部资料、拒绝越权、不泄露 secret、工具调用前查权限、说明拒绝原因 |
| **Key / Secret Isolation** | 私钥/API key/密码不进模型上下文。Agent 请求受控动作而非拿秘密本身：请求"查询余额"而非拿 RPC key、请求"生成交易草稿"而非拿私钥 |
| **Privacy-preserving Workflow** | 只读必要数据、外部模型调用做脱敏摘要、默认隐藏不必要地址/金额/身份、展示授权范围、允许查看 Agent 用了哪些信息 |
