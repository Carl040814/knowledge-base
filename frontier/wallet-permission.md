# [钱包与权限（Wallet / Permission）](https://aiweb3.school/zh/handbook/tracks/wallet-permission)

> 约 7 分钟阅读 · 2026-05-12

**核心原则**：AI Agent 一旦能操作钱包，产品问题从"能不能完成任务"变成"它到底被允许做什么"。钱包不是签名弹窗，是 Agent 执行能力的权限边界。

## 第一性原理

Agent 不应该拥有"钱包"，只应该拥有一组可限制、可审计、可撤销的能力。

- **权限要比意图更具体**：不要"帮我管理资金"，要限制资产/额度/合约/函数/时间/频率
- **签名前要可理解**：用户看到的不是 calldata，而是资产变化/授权对象/最大损失/失败路径
- **撤销是默认能力**：任何长期权限都应能被用户查看、暂停和撤销

## 知识节点

| 节点 | 说明 |
|------|------|
| **AI Wallet UX** | 展示：用户目标、行动计划、权限请求（资产/合约/函数/时间）、交易解释（签名后资产变化）、风险提示（最坏损失）、撤销入口。重点不是少点确认，而是让确认有意义 |
| **Permission Policy** | 系统可执行规则：资产范围、金额上限（单笔/日/周/总）、目标合约白名单、函数范围、滑点阈值、时间窗口、频率限制。AI 生成草稿，钱包/智能账户/Guard 校验 |
| **Session Key Flow** | 用户创建 session key→设权限范围→Agent 用 key 发起合规操作→系统校验 policy→到期/超限/撤销后失效。价值是减少重复签名，不能绕过风险控制 |
| **Safe Guard** | 交易执行前确定性检查器：目标地址白名单、calldata 函数是否允许、approval 是否超限、滑点是否过大、接收地址是否合法、模拟结果是否符合预期、频率是否异常 |
