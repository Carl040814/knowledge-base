# [治理（Governance - 前沿）](https://aiweb3.school/zh/handbook/tracks/governance)

> 约 6 分钟阅读 · 2026-05-12

**核心原则**：AI 可降低治理参与的信息成本，但也能把复杂争议压扁成看似客观的摘要。治理 Agent 的重点不是替人投票，而是让人更清楚地理解、讨论和执行公共决策。

## 第一性原理

治理 Agent 不能替代公共判断，只能降低理解和协作成本。

- **摘要必须可追溯**：每个关键判断回原文、会议或链上记录
- **不同立场要被保留**：治理不是消除分歧，是让分歧更清楚
- **执行要可核对**：提案通过后预算/任务/负责人/链上执行都要可追踪

## 知识节点

| 节点 | 说明 |
|------|------|
| **Proposal Summarizer** | 长提案→背景/目标/预算/执行步骤/风险/支持理由/反对理由。质量标准是来源保真，不是文风流畅 |
| **Meeting-to-Action** | 会议→决定、未共识问题、每个 action 负责人+截止时间、需提交论坛/投票/多签事项、下次会议前准备材料。区分"有人提到"和"已经决定" |
| **Contribution Tracker** | 记录 GitHub/论坛/链上交易/资助/会议/交付物。不只按活跃度排名：代码审计、文档翻译、社区组织、安全修复价值不同 |
| **Budget Checklist** | AI 审查：预算总额+资产、分阶段付款条件、交付物+验收方式、负责人+收款地址+利益冲突、失败/延期处理、历史预算对比 |
| **Public-goods Impact Dashboard** | 多证据聚合：项目产出、使用情况、生态影响、维护情况、资金使用。目的不是替投票者决定，是减少信息收集时间 |
| **Source-preserving Summary** | 底线能力：结论→证据（原文段落/会议时间戳/链上记录）→链接→不确定性标注。区分事实和 AI 推断 |
