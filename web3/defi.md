# [去中心化金融（DeFi）](https://aiweb3.school/zh/handbook/web3/defi)

> 约 5 分钟阅读 · 2026-05-12

**核心原则**：DeFi 是建立在智能合约上的开放金融系统。它让资产、交易、借贷、稳定币和流动性可以被协议化组合，也让风险沿着协议依赖链传播。

DeFi 的核心不是"去掉中介"，而是把金融规则变成可组合、可验证、也可被攻击的链上协议。

## 第一性原理

DeFi 协议管理的是资产状态，不只是应用界面。

传统金融里很多状态在机构账本里。DeFi 把余额、抵押、债务、流动性、交易和清算放进合约系统。透明度提高了，但所有机制错误、价格冲击和外部依赖也会更快暴露。

- **资产是协议输入**：token 标准、decimals、授权和余额都会影响协议行为
- **流动性决定可执行性**：有价格不等于能成交，能成交不等于低成本成交
- **可组合性放大风险**：一个协议出问题，可能影响依赖它的多个协议

## 知识节点

| 节点 | 解释 |
|------|------|
| **Token** | DeFi 的资产单位。ERC-20 最常见。看 token 时检查：合约地址是否正确、decimals、总供应量/发行权限、是否可暂停/冻结/增发/升级、是否有转账税或黑名单。错误的地址和 decimals 是常见风险源 |
| **AMM** | 用流动性池和定价公式替代订单簿（如 Uniswap）。核心概念：滑点（预期 vs 实际价差）、流动性提供者（赚手续费但承担无常损失）、价格影响（大单改变池子比例）、MEV 风险（sandwich attack） |
| **Lending** | 借贷协议把存款、借款、抵押率、利率和清算规则写进合约。风险：抵押品价格快速下跌、预言机延迟、流动性不足清算失败、参数过于激进 |
| **Stablecoin** | DeFi 计价和结算基础。不同机制（法币储备/加密超额抵押/算法/混合）有不同风险：储备透明度、赎回能力、抵押品波动、治理参数、脱锚时谁承担损失 |
| **Liquidity** | 决定资产能否以合理价格买卖/借出/清算/退出。没有流动性支撑的价格只是屏幕上的数字 |

## 在 AI × Web3 中的位置

AI 进入 DeFi，最适合先做信息整理和风险辅助：解释交易、总结仓位、监控清算、识别异常价格。真正高风险的是自动执行：swap、借贷、加杠杆、授权、跨链和清算。

Agent 执行 DeFi 操作至少要有：额度限制、协议白名单、滑点上限、模拟结果、预言机检查、人工确认和交易后审计记录。

## 最小实践

1. 选一个公开的 swap 交易哈希
2. 在区块浏览器查看输入/输出 token、路由、滑点、手续费
3. 找到涉及的池子或路由合约
4. 解释用户实际收到多少和预期价格差距
5. 写出让 AI Agent 执行同类交易必须设置的限制
